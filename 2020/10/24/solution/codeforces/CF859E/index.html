[连通块][思路] CF859E Desk Disorder 题解

本文最后更新于：2020年10月24日 上午

CF859E Desk Disorder 题解

题面：CodeForces 洛谷

题目大意

现在有 2N2N2N 个节点和 NNN 个人，每个人现在都处于一个节点上 ( 保证没有两个人同于一个节点 )，且每个人都有自己想去的节点 ( 可以是当前他自己所在的节点 )，一个人可以保持在原节点不动，也可以去自己想去的节点，问保证不存在一个节点上有超过一个人的方案数有多少种 ( 对 109+710^9+7109+7 取模 )。

解法

考虑将想去节点的关系建成一张图，则这张图上有多个连通块，且它们的答案互不影响，所以分别计算每个连通块的答案，并用乘法原理统计最终答案。

根据题意，可以发现只存在出度为 000 ( 没有人 ) 和出度为 111 ( 有人 ) 的节点，则在同一个连通块中至多只会存在 111 个出度为 000 的点。

假设没有自环，讨论这样的点存在与否：

  1. 有一个这样的点：

    可以发现，这种情况形成的连通块一定是树，我们以这个无出度的点作为树根，如下图所示：

    这样，对于树上的一个节点，如果它要到达下一个节点，它所指向的节点这一路径上的所有点都要前进一步，而每一个节点都有一个这样的方案，故方案为 nnn ( nnn 为树的大小 )[1]。

  2. 不存在这样的点

    这时候，这个连通块会变成一个环，或一个基环树，如下图所示：

    此时，对于一个环，只有全部移动和全部不动两种情况，而对于一个基环树，无论环上如何运动，每一棵树上的其他点都是不可动的，故这两种情况的方案均为 222。

最后，由于题目中可能有自环，而存在自环的连通块只存在保持原位置不变一种合法情况[2]，故方案为 111。

所以，采用并查集判环便可解决本题。

代码

# include <iostream>
# include <cstdio>
# define MAXN 100005

constexpr long long MOD = 1e9+7;

int fa[MAXN<<1];
bool selfLoop[MAXN<<1], outDeg[MAXN<<1], loop[MAXN<<1];
long long siz[MAXN<<1];

int Find(int x){
	return fa[x] == x ? x : fa[x] = Find(fa[x]);
}

int main(){
	int n; long long ans = 1;
	scanf("%d", &n);

	for(int i = 1; i <= n * 2; i++){
		fa[i] = i, siz[i] = 1;
	}

	for(int i = 1, x, y; i <= n; i++){
		scanf("%d%d", &x, &y);
		outDeg[x] = true;
		if(x == y){
			selfLoop[x] = true;
			continue;
		}
		else{
			x = Find(x), y = Find(y);
			if(x == y && !loop[x]){
				ans = (ans * 2ll) % MOD;
				loop[x] = true;
				continue;
			}
			siz[y] += siz[x], selfLoop[y] |= selfLoop[x];
			fa[x] = y;
		}
	}

	for(int i = 1; i <= 2 * n; i++){
		if(!outDeg[i] && !selfLoop[i]){ // 没出度的点 loop 一定为 false
			ans = (ans * siz[i]) % MOD;
		}
	}

	printf("%lld", ans);

	return 0;
}
  1. n−1n-1n−1 种方案是除树根的每个节点移动一次，还有一种是都不动。 ↩
  2. 相当于是一个环大小为 111 的基环树，它的环上两种方案是等价的，故算作一种。 ↩
